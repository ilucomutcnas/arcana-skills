# Skill Catalog

This file maps each mini-skill to its reference and examples file.

## 1. foundations
- Summary: Defines the visual and semantic foundation of a design system: tokens, scales, primitives, and naming.
- Reference: `references/skills/foundations.md`
- Examples: `examples/skills/foundations.md`

## 2. components
- Summary: Designs reusable component contracts, states, variants, and composition rules for a scalable UI library.
- Reference: `references/skills/components.md`
- Examples: `examples/skills/components.md`

## 3. theming
- Summary: Defines multi-theme, light/dark, and multi-brand theming with semantic safety and implementation discipline.
- Reference: `references/skills/theming.md`
- Examples: `examples/skills/theming.md`

## 4. accessibility
- Summary: Bakes accessibility into tokens, components, theming, and documentation at the system level.
- Reference: `references/skills/accessibility.md`
- Examples: `examples/skills/accessibility.md`

## 5. documentation
- Summary: Documents system intent, usage rules, component contracts, and adoption guidance for design and engineering teams.
- Reference: `references/skills/documentation.md`
- Examples: `examples/skills/documentation.md`

## 6. governance
- Summary: Defines contribution, review, versioning, deprecation, and ownership rules for sustainable design systems.
- Reference: `references/skills/governance.md`
- Examples: `examples/skills/governance.md`

## 7. review-audit
- Summary: Audits products or libraries against design-system rules and produces concrete remediation guidance.
- Reference: `references/skills/review-audit.md`
- Examples: `examples/skills/review-audit.md`

## 8. framework-mapping
- Summary: Maps design-system concepts to CSS, Tailwind, CSS Modules, utility-first systems, and CSS-in-JS without losing semantic intent.
- Reference: `references/skills/framework-mapping.md`
- Examples: `examples/skills/framework-mapping.md`
