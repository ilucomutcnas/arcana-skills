# Anti-Patterns

- Mixing public API docs with internal contributor heuristics in the same mini-skill.
- Turning exploratory notes into mandatory rules without evidence.
- Dropping browser-specific caveats because they seem verbose.
- Making performance claims without refreshing the relevant snapshots.
- Changing the hot `layout()` path while reasoning from demo convenience.
- Overfitting one browser, one width, or one extractor-sensitive miss.
- Reopening solved corpora because a new canary feels more interesting.
- Removing old tasks instead of preserving them in a cleaner routing structure.
