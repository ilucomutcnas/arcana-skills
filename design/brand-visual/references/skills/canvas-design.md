## Purpose
Create design philosophies — aesthetic movements expressed through code. Each creation involves two steps: defining a design philosophy as a document, then expressing it visually on a canvas as a .pdf or .png artifact. A library of 81 curated fonts is available for typography selection.

## When to Use
- When creating visual art, poster designs, or static visual pieces
- When a design philosophy must drive the visual expression
- When producing museum-quality visual artifacts via code-generated canvas output
- When curated typography selection from a font library is needed

## Workflow
1. Create a design philosophy document (.md) explaining the aesthetic movement, its principles, and visual language.
2. Express the philosophy visually by creating it on a canvas (.pdf or .png).
3. Use fonts from `canvas-design__canvas-fonts/` for typography.
4. Refine the work — do not add more graphics; instead make the existing composition more cohesive.
5. Take a second pass to polish the output into a philosophically designed masterpiece.

## Quality Standards
- The output must feel pristine, like a museum piece.
- Refine rather than add — make existing elements more cohesive.
- If the instinct is to add a new shape or effect, stop and ask: "How can I make what is already here more of a piece of art?"
- Respect the design philosophy and principles of minimalism.

## Technical Rules
- Output formats: .md (philosophy), .pdf or .png (visual artifact).
- Fonts available in `canvas-design__canvas-fonts/` — includes serif, sans-serif, mono, and display families.
- Code-generated canvas output — use Python libraries (Pillow, ReportLab, etc.) or HTML canvas.

## Validation
- Verify the visual artifact expresses the design philosophy.
- Check that typography choices are appropriate for the aesthetic movement.
- Confirm the composition is cohesive and refined, not cluttered.

## Restrictions
- Do not add more graphics to fix a composition — refine what exists.
- Do not skip the second refinement pass.
- Do not create the visual without first defining the philosophy.

## Output Requirements
- Design philosophy as markdown document.
- Visual artifact as .pdf or .png file.
- Both outputs must be philosophically aligned.
