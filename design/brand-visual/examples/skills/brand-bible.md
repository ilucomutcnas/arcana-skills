# Brand Bible — Examples

## Tone by Context

| Context | Tone | Personality Level |
|---------|------|-------------------|
| Error messages | Empathetic, solution-oriented | Low |
| Settings pages | Neutral, clear | None |
| Documentation | Informative, structured | None |
| Billing flows | Trustworthy, precise | None |
| Empty states | Warm, encouraging | Medium |
| 404 pages | Self-aware, fun | High |
| Loading states | Light, brief | Low |

## Copy Examples

```
"This page doesn't exist. Maybe it never did. Maybe it was a dream.
 Either way, let's get you back on track."

"No errors yet. Enjoy this moment of peace while it lasts."

"Let's get your first error. Don't worry, it's not as scary as it sounds."

"Crunching the numbers..."

"Fetching your data..."
```

## Writing Principles

1. Be concise — fewest words needed.
2. Be direct — tell users what to do, not what they can do.
3. Use active voice — "Save your changes" not "Your changes will be saved".
4. Avoid jargon — simple words users understand.
5. Be specific — "3 errors found" not "Some errors found".
